# Citation & Hallucination Checker — 引文与主张验证

双轮架构验证工具。验证每一条主张：引用是否存在、来源是否说了声称的内容、数值是否精确。

**设计原则**：确定性验证。相同输入 → 相同输出。

---

## 两种验证模式

### 模式 1：Search 验证（默认）
- 在 Web 上搜索权威来源
- 验证引用是否真实存在
- 检查引用来源是否真的说了声称的内容
- 为统计数据找到原始来源

### 模式 2：Doc-Only 验证
- 用户提供源文档
- 所有主张必须能追溯到该文档
- 标记任何看似来自外部知识的内容
- 触发词："只用这篇文档验证" / "不搜索网络" / "仅对照原文"

---

## 双轮架构

**关键**：始终用两个独立轮次。绝不在提取时同时验证。

### Pass 1：仅提取

1. 阅读整个文档（或用户输入的文本）
2. 按照主张提取规则（见下）提取所有主张
3. 输出编号列表：`[claim_id] | [主张文本] | [类型] | [位置]`
4. **此轮不做任何验证**
5. 将提取结果呈现给用户确认后再进行 Pass 2

### Pass 2：仅验证

1. 以 Pass 1 的输出为固定输入
2. 按顺序逐条验证每个 claim_id
3. **不允许重新提取**——仅基于 Pass 1 的主张
4. 对每条主张应用状态决策树
5. 生成最终报告

## 主张提取规则

### 需要提取的类型

| 类型 | 模式 | 示例 |
|------|------|------|
| **统计数据** | 任何带单位的数字 | "92.3% 准确率"、"47亿美元市场" |
| **比较性** | X 比 Y 更… | "比基线快3倍" |
| **时间性** | 时间限定的断言 | "2024年，采用率达到…" |
| **归因性** | 与来源关联的主张 | "根据 WHO…"、"Smith等人发现…" |
| **因果性** | X 导致/引起 Y | "这将延迟减少…" |
| **存在性** | 断言某事物存在/为真 | "有5亿用户"、"模型支持…" |
| **排序性** | 位置主张 | "最大的"、"首个"、"前三" |
| **引语** | 直接引用 | 引号内归因于来源的文本 |

### 不需要提取的类型

| 类型 | 原因 |
|------|------|
| 定义 | "机器学习是 AI 的一个子集"——定义性，非事实主张 |
| 已标记的主观意见 | "我们认为…"、"我们看来…"——明确主观 |
| 假设性陈述 | "如果采用继续…"、"可能…"——推测 |
| 疑问句 | "是什么推动了增长？"——非断言 |
| 无来源的未来预测 | "到2030年将达到100亿"——除非引用预测报告的来源 |
| 方法论描述 | "我们使用 PyTorch 2.0"——过程，非事实主张 |
| 致谢 | "感谢我们的合作者"——不可验证 |

### 提取输出格式

```
[C01] | "模型在 ImageNet 上达到 96.555% 准确率" | 统计数据 | 第3页，第2段
[C02] | "在推理任务上比 GPT-4 高出 12%" | 比较性 | 第5页，第1段
[C03] | "根据 Chen et al. (2024)，transformers 呈线性扩展" | 归因性 | 第5页，第2段
[C04] | "2024年市场规模达到47亿美元" | 统计数据+时间性 | 第7页，图表标题
```

## 状态决策树

对每条主张严格按以下决策树操作：

```
START
│
├─ 这是引用主张（引用了某篇论文/报告/来源）？
│   ├─ 是 → 进入"引用验证"
│   └─ 否 → 进入"统计/事实验证"
│
│
引用验证（CITATION VALIDATION）
│
├─ Step 1：被引用的来源是否存在？
│   │   运行全部强制搜索查询（见搜索模板）
│   │
│   ├─ 否 → 状态："引用未找到"
│   │        问题："无法在任何数据库中定位 [引用]"
│   │        停止
│   │
│   └─ 是 → Step 2：来源是否包含声称的话题？
│             │
│             ├─ 否 → 状态："误引用"
│             │        问题："来源存在但不讨论 [话题]"
│             │        停止
│             │
│             └─ 是 → Step 3：来源是否支持确切主张？
│                       │
│                       ├─ 是（完全匹配）→ 状态："已验证"
│                       │                   置信度："精确"
│                       │
│                       ├─ 是（改写，同义）→ 状态："已验证"
│                       │                     置信度："改写"
│                       │
│                       ├─ 部分（缺上下文）→ 状态："误导"
│                       │   问题："主张省略了关键上下文：[缺失的内容]"
│                       │
│                       └─ 否（矛盾）→ 状态："幻觉"
│                           问题："来源说 [X]，主张说 [Y]"
│
│
统计/事实验证（STATISTIC/FACT VALIDATION）
│
├─ Step 1：能否找到权威来源？
│   │   运行全部强制搜索查询
│   │
│   ├─ 否（找不到任何来源）→ 状态："未验证"
│   │                         问题："未找到权威来源"
│   │                         停止
│   │
│   └─ 是 → Step 2：数值是否完全匹配？
│             │
│             ├─ 是 → 状态："已验证" 置信度："精确"
│             │        停止
│             │
│             └─ 否 → 状态："数值错误"
│                     进入"数值错误详细说明"
│
│
数值错误详细说明（NUMERICAL ERROR DETAILS）
│
├─ 记录：
│   • 来源数值：[来源中的确切数字]
│   • 声称数值：[被查文档中的数字]
│   • 偏差：[计算确切差异]
│   • 来源位置：[页、表、节]
│
├─ 分类：
│   • 任何四舍五入 → 数值错误
│   • 任何截断 → 数值错误
│   • 有效数字不匹配 → 数值错误
│   • 单位不匹配 → 数值错误
│   • 方向错误（如增长 vs 减少）→ 幻觉
│
└─ 例外：如果来源自身提供了四舍五入的数字
    • 例如，来源说 "96.555%（约97%）"
    • 则声称 "97%" → 已验证（引用近似值）
```

## 学术精度数值规则

**默认模式：严格学术精度。仅接受确切数字。**

| 规则 | 来源 | 声称 | 状态 |
|------|------|------|------|
| 必须完全匹配 | 96.555% | 96.555% | ✓ 已验证 |
| 任何四舍五入=错误 | 96.555% | 97% | ✗ 数值错误 |
| 任何四舍五入=错误 | 96.555% | 96.6% | ✗ 数值错误 |
| 截断=错误 | 96.555% | 96.5% | ✗ 数值错误 |
| 有效数字必须匹配 | 0.834 | 0.83 | ✗ 数值错误 |
| 单位必须匹配 | 96.555% | 0.96555 | ✗ 数值错误 |
| 方向重要 | +12% 增长 | +15% 增长 | ✗ 幻觉 |
| 数量级重要 | 47亿美元 | 470亿美元 | ✗ 幻觉 |

## 强制搜索模板

对每种主张类型，运行所有适用的搜索查询。找到第一个结果后不停止。

### 对学术引用

```
Query 1: "[第一作者姓氏] [年份] [标题前3个词]"
Query 2: "[完整论文标题]" site:semanticscholar.org OR site:arxiv.org
Query 3: "[第一作者] [年份] [期刊/会议名称]"
Query 4: "doi:[DOI]"（如有DOI）
Query 5: "arxiv:[arxiv_id]"（如有arXiv ID）
```

### 对统计数据

```
Query 1: "[带单位的精确数字] [话题] [年份]"
Query 2: "[话题] [年份] 统计报告"
Query 3: "[话题] [年份] 报告 site:gov OR site:edu"
Query 4: "[话题] 市场规模 [年份]"
```

### 对公司/产品主张

```
Query 1: "[公司名] [主张话题] 新闻稿 [年份]"
Query 2: site:[公司域名] [主张话题]
Query 3: "[公司名] [指标] 官方公告"
```

### 对健康/医学主张

```
Query 1: "[主张话题] site:who.int OR site:cdc.gov OR site:nih.gov"
Query 2: "[主张] 系统综述 site:pubmed.ncbi.nlm.nih.gov"
```

## 来源权威层级

当找到多个来源时，按此顺序优先：

| 排名 | 来源类型 | 示例 |
|------|---------|------|
| 1 | 原始来源 | 原始研究、官方报告、原始数据 |
| 2 | 政府/机构 | WHO、CDC、世界银行、国家统计局 |
| 3 | 同行评议出版物 | Nature、Science、IEEE、ACM |
| 4 | 具名行业报告 | Gartner、McKinsey、Statista（含方法论说明） |
| 5 | 引用原始来源的可靠新闻 | NYT、Reuters 引用原始来源 |
| 6 | 二次汇编 | Wikipedia（查其来源） |

**规则**：如果只找到排名 5-6 的来源，状态 = "未验证"，标注"仅找到二次来源"。

---

## 多源验证

| 条件 | 所需来源 |
|------|---------|
| 找到原始来源 | 1（如果是权威的：.gov、同行评议、官方） |
| 仅二次来源 | ≥2 个独立来源一致 |
| 来源冲突 | 状态 = "未验证"，记录冲突 |

---

## 裁决规则

| 情况 | 规则 |
|------|------|
| 主张缺日期 | 假设指最新可用年份；标记"需补充日期" |
| 来源冲突 | 用最新权威来源；引用两者；记录冲突 |
| 所有查询后找不到来源 | 状态 = "未验证"（非"幻觉"） |
| 数字因货币换算不同 | 标记"需澄清：币种/单位" |
| 同一机构多份报告 | 用最新；注明日期 |
| 主张用了"约"/"大约" | 仍验证基数是否在合理范围（±10%） |
| 来源付费墙 | 注明"来源需付费，无法验证确切文本" |
| 来源为外文 | 翻译后验证；注明翻译 |

## Doc-Only 模式工作流

**触发词**："只用这篇文档验证" / "不搜索网络" / "仅对照原文"

### Step 1：建立源文档索引

```
SOURCE INDEX
文档：[文件名]
页数：[N]

第1页：
- 文本：[内容摘要]
- 统计数据：[列出所有带上下文的数字]
- 表格：[表1：列X, Y, Z]
- 图：[图1：展示X]
...
```

### Step 2：应用双轮架构

同 Search 模式，但仅用源文档索引验证。

### Step 3：追溯每条主张

```
主张：[C01] "模型达到92%准确率"
搜索索引："92"、"准确率"、"性能"
├─ 已找到：第4.2节，第8页 — "Our model achieves 92.1% accuracy"
│   └─ 状态：数值错误（92% vs 92.1%）
OR
├─ 索引中未找到
│   └─ 状态："不在来源中"
│       问题："此主张无法追溯到提供的文档"
│       可能：外部知识/幻觉
```

### Step 4：标记所有外部知识

Doc-Only 模式下，任何无法追溯到来源的主张 = 问题。

## 关键规则

1. **始终双轮**——先提取，后验证。绝不交叉。
2. **每条主张都要检查**——无例外，不跳"明显"的。
3. **仅接受确切数字**——96.555% ≠ 97%（学术模式）。
4. **找到原始来源**——不接受未知原始来源的二次引用。
5. **运行所有搜索模板**——不在第一个结果后停止。
6. **引用必须真实**——搜索确认论文/报告存在。
7. **检查来源实际说了什么**——真实论文也可能被误引。
8. **Doc-Only 模式下标记所有外部知识**——即使它是正确的。
9. **不确定时倾向保守**——"未验证"比错误的"已验证"更安全。
10. **遵循裁决规则**——不做临时判断。
