# Literature Survey — 结构化文献调研

基于 Kahneman 双过程理论、Keshav 三遍法、第一性原理分析。所有结果写入本地 Obsidian/文件系统，搜索后端用 WebSearch + WebFetch。

**模式**：
- `/survey intent` — 明确调研目标和起点
- `/survey triage` — 快速筛选，绘制领域全景图
- `/survey deepen` — 深度阅读，提取写作工艺和可视化分析
- `/survey synthesize` — 跨论文综合分析（Related Work / Gap Analysis / 新 Idea）
- `/survey expand` — 语料扩展建议

---

## 模式 1：Intent — "你要搞清楚什么？"

**目的**：在阅读任何论文前，明确调研目标、已有知识水平和成功标准。

### Step 1：识别用户原型

> 在开始找论文之前，先搞清楚这是什么类型的调研。以下哪种最接近你的情况？
>
> **A. 探索者** — "我刚进入一个新领域，需要理解全局。" 
> **B. 调查者** — "我有几个具体问题，需要从文献中找到答案。"
> **C. 验证者** — "我觉得我发现了一个 gap/idea，想确认它是否真的新颖。"
> **D. 考官** — "我需要展示对该领域的全面掌握（开题/综述论文/资格考试）。"

| 原型 | 筛选规模 | 深度阅读目标 | 深度篇数 | 综合输出 |
|------|---------|-------------|---------|---------|
| 探索者 | 50-100+ 篇 | Pass 2: 8-15篇 | 2-3 | 领域全景 |
| 调查者 | 10-25 篇 | Pass 2: 5-10篇 | 3-5 | 技术对比 |
| 验证者 | 15-30 篇 | 最接近的3-5篇 Pass 2+3 | 3-5 | 定位论证 |
| 考官 | 80-150+ 篇 | Pass 2: 15-25篇 | 5-8 | 完整综述叙事 |

### Step 2：校准已有知识

> 1. **术语检查**：你能说出3个该领域的关键术语并各用一句话定义吗？
> 2. **里程碑论文**：你能说出哪些相关的论文、作者或课题组？
> 3. **当前心智模型**：用2-3句话描述你对主要问题和方法的目前理解。
> 4. **已知的未知**：你希望文献能回答哪些具体问题？

**认知目的**：建立基线，之后对比知识变化。

### Step 3：定义成功标准

> 1. **交付物**：Related Work 章节？开题报告？Gap Analysis？Idea 验证？
> 2. **规模**：大约需要覆盖多少篇论文？
> 3. **时间线**：什么时候需要完成？
> 4. **时间预算**：每周能投入多少小时？

### Step 4：生成意图档案

在 Obsidian 中创建笔记：`literature-survey/<topic-slug>/survey_intent.md`

包含：原型、校准答案、成功标准、已知的来源/课题组。

## 模式 2：Triage — "这堆论文里有什么？"

**目的**：对一批论文进行快速 Pass 1（Keshav 三遍法第一遍），绘制领域全景图，决定哪里值得投入深度阅读。

### Step 1：确定搜索源

根据用户原型和主题，选择搜索策略：
- **新领域探索** → Google Scholar 被引排序 + 最新综述
- **特定问题** → 关键词组合搜索（见 battery-search 模板）
- **查漏补缺** → Semantic Scholar 引文追踪

### Step 2：批量 Pass 1（每篇 5-10 分钟）

对每篇论文提取：
| 维度 | Pass 1 内容 |
|------|-----------|
| 类别 | 实验测量 / 系统构建 / 理论 / 综述 |
| 问题 | 一句话核心矛盾 |
| 贡献 | 主要声称（尽量引用原文） |
| 评估方式 | 数据集/测试体系/对照组 |
| 关键参考文献 | 3-5 个最常引用的 |
| 与调研相关性 | 高/中/低 |

用 WebSearch 搜索每篇论文的摘要和标题，用 WebFetch 获取详细摘要页面。

每篇论文写到一个本地 markdown 文件中标注 `[Pass 1]`。

### Step 3：生成全景图

综合所有 Pass 1 结果，回答：
1. 这个领域有哪些主要研究方向（3-5条线索）？
2. 每条线索的代表性论文和方法论是什么？
3. 有什么时间上的演变规律？
4. **WYSIATI 检查**：哪些问题/方法在这个语料中缺失？一个怀疑论的审稿人会说缺了什么？

输出到 `survey_triage.md`。

### Step 4：WYSIATI 对比

> "这是你开始时以为领域长这样的样子（从 intent 档案）。这是语料实际呈现的样子。有什么不同？什么让你惊讶？"

### Step 5：优先排序阅读深度

- **仅 Pass 1**：背景知识
- **Pass 2 推荐**：核心线索相关
- **Pass 3 必需**：基础性论文，必须深度理解

输出到 `survey_triage.md`，标注优先级。

### Step 6：发现扩展候选

- 哪些参考文献在 3+ 篇论文中被引用但不在当前语料中？
- 对每个候选，标注原因：基础性/依赖性/填补空缺/对立观点/邻近领域
- 列出到 `corpus_log.md`

## 模式 3：Deepen — "这篇论文到底说了什么？"

**目的**：对单篇论文进行 Pass 2 和 Pass 3 深度阅读。强制 System 2 参与，本地记录洞察。

### Step 1：选择论文

从 triage 阅读列表中选择一篇。

### Step 2：Pass 2 协议（四轮查询）

对每篇深度阅读论文，通过 WebSearch/WebFetch 获取以下信息并写入本地笔记：

**a. 主张提取**：引用该论文最重要的3个主张，附章节/页码。用 WebFetch 获取引言和结论部分。

**b. 证据审计**：对每个主张，提供的证据是什么？评级：强/中/弱。

**c. 方法论探测**：评估体系是什么？显式和隐式假设？如果负载/规模/拓扑变了，什么会崩溃？

**d. 依赖提取**：该论文依赖哪些其他论文的结果？这些依赖在别的场景下可能不成立吗？

写入本地笔记 `[Pass 2]` 章节。

### Step 3：校准检查

1. 先问用户："用一句话说，这篇论文的主要贡献是什么？"
2. 然后用 WebFetch 确认："引用摘要或引言中的确切主要贡献声明。"
3. 两者并排展示，高亮差距。写入 `[Calibration]` 章节。

### Step 4：第一性原理分解

从四个维度分析：
- **STATE**：系统管理了什么状态？
- **TIME**：什么时间尺度起作用？
- **COORDINATION**：组件之间如何协调？
- **INTERFACE**：组件之间的边界是什么？

写入 `[First-Principles]` 章节。

### Step 5：Pass 3 — 虚拟复现（仅基础性论文）

- "如果你必须从零构建这个系统（同样目标），你的设计会是什么样？"
- "哪些假设从未被明确陈述？"
- "写三个一个怀疑的审稿人会问的关键问题。"

### Step 6：写作工艺提取（用户欣赏的论文）

**a. 引言解剖——六步公式**：
通过 WebFetch 获取引言全文，分析：
1. MOVE 1 (Stakes)：怎么开头的？具体行动者/应用/数字？
2. MOVE 2 (Problem Gap)：结构化还是量化的？编号列出局限性？
3. MOVE 3 (Key Abstraction)：是否创造了一个可引用的记忆点？
4. MOVE 4 (Design Intuition)：一段话的心智模型？概览图？
5. MOVE 5 (Contributions)：带证据的主张，还是过程描述？编号？
6. MOVE 6 (Results Preview)：具体的头条数字？

**b. Evaluation 架构**：
- 主张-证据映射：每个引言主张 → Evaluation 小节 → 图/表
- 设置：压缩版还是技术报告？
- 深度：结果按有意义的维度分拆？
- 每个实验簇后有明确的 Takeaway？
- Ablation：展示每个组件的贡献？

写入 `[Craft]` 章节。关键经验追加到 `synthesis/writing_craft_corpus.md`。

### Step 7：可视化提取（所有 Pass 2+ 论文）

**a. 图表清单**：列出每张图和表。对每张：标题、角色（概览/对比/深度/ablation/案例）、支持的主张、编码方式。哪张是头条图？

**b. 视觉论证分析（2-3张关键图）**：
- 每张图支持什么主张？
- 编码选择描述（轴、尺度、颜色、分面）
- 为什么选这些编码——编码服务于论证吗？
- 该图没有展示什么有用的东西？

写入 `[Visualization]` 章节。

### 输出结构

每篇论文的本地笔记增长路径：
`[Pass 1]` → `[Pass 2]` → `[Calibration]` → `[First-Principles]` → `[Craft]` → `[Visualization]` → `[My Notes]`

## 模式 4：Synthesize — "所有这些论文之间有什么联系？"

**目的**：跨论文综合分析，产出具体交付物。

### Step 1：选择综合目标

- **Related Work 章节**：有组织的主题叙事
- **开题/资格考试**：广度 + 深度 + 前沿识别
- **Research Gap 识别**：系统化分析缺失
- **新 Idea 合成**：创造性重组

### Step 2：不变性矩阵

对所有论文构建对比矩阵：
| 论文 | 状态管理 | 主要时间尺度 | 协调模型 | 接口设计 |
|------|---------|------------|---------|---------|

高亮论文之间做出根本不同选择的地方。

写入 `synthesis/invariant_matrix.md`。

### Step 3：依赖图

识别：一篇论文的设计依赖于另一篇论文挑战的假设。引用具体假设和挑战证据。

写入 `synthesis/dependency_graph.md`。

### Step 4：Gap 识别

> "如果带宽/延迟/规模变化 10 倍，哪些方案仍然有效？哪些崩溃？出现了什么新问题但没有论文解决？"

> 查看不变性矩阵——有没有组合没有论文探索？缺失的方法论？

写入 `synthesis/gap_analysis.md`。

### Step 5：叙事构建

根据综合目标生成：
- **Related Work**：主题线索配知识演进弧线，非编年列表。每条线索："这些论文通过解决 Y 来处理 X，但都没有处理 Z——我们的贡献。"
- **开题/资格考试**：子领域广度 + 2-3篇基础论文深度 + 前沿识别 + 学生自身定位。
- **Gap Analysis**：约束变化分析 → 候选问题 + 第一性原理论证。

### Step 6：WYSIATI 最终审计

> "缺少了什么视角？邻近领域的人会怎么说？我是否过度索引了某一个课题组/会议/方法论？"

## 跨模式：Expand — "语料该扩展吗？"

**触发时机**：triage 中常引用的不在语料的论文；deepen 中未读的依赖项；synthesize 中发现的整个缺失子领域。

**流程**：
1. 批量候选：列出候选论文、发现来源、原因、与意图相关性、建议 Pass 级别、时间成本
2. 用户选择：Add（加入语料）/ Bookmark（暂存 backlog）/ Skip（记录原因）
3. **预算守卫**：跟踪扩展对意图档案中范围和时间预算的影响

> "你最初目标是约30篇论文。现在已经是42篇了。剩余时间预算仅支持深读约5篇。调整范围还是要更挑剔？"
